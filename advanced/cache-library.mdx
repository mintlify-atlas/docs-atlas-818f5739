## Overview

The RuneLite cache library provides comprehensive tools for reading and writing Old School RuneScape's game cache. The cache contains all game assets including items, objects, NPCs, maps, models, sprites, and scripts.

## Cache Structure

### Store and Index System

The cache is organized into a hierarchical structure:

```
Store (main cache directory)
  └─ Index (e.g., ITEMS, MODELS, SPRITES)
       └─ Archive (groups of related data)
            └─ Files (individual data entries)
```

### Opening a Cache

```java
import net.runelite.cache.fs.Store;
```

## Working with Archives

### Retrieving Archive Data

```java

// Get an archive from an index
```

## Container Format

Archives are stored in containers with compression and integrity checking:

```java
// From Container.java:93-172
public static Container decompress(byte[] b, int[] keys) throws IOException
{
    InputStream stream = new InputStream(b);
    
    int compression = stream.readUnsignedByte();
    int compressedLength = stream.readInt();
    
    Crc32 crc32 = new Crc32();
    crc32.update(b, 0, 5); // compression + length
    
    byte[] data;
    switch (compression)
    {
        case CompressionType.NONE:
        {
            byte[] encryptedData = new byte[compressedLength];
            stream.readBytes(encryptedData, 0, compressedLength);
            
            crc32.update(encryptedData, 0, compressedLength);
            byte[] decryptedData = decrypt(encryptedData, encryptedData.length, keys);
            
            data = decryptedData;
            break;
        }
        case CompressionType.BZ2:
        {
            byte[] encryptedData = new byte[compressedLength + 4];
            stream.readBytes(encryptedData);
            
            crc32.update(encryptedData, 0, encryptedData.length);
            byte[] decryptedData = decrypt(encryptedData, encryptedData.length, keys);
            
            InputStream decryptedStream = new InputStream(decryptedData);
            int decompressedLength = decryptedStream.readInt();
            data = BZip2.decompress(decryptedStream.getRemaining(), compressedLength);
            break;
        }
        case CompressionType.GZ:
        {
            byte[] encryptedData = new byte[compressedLength + 4];
            stream.readBytes(encryptedData);
            
            crc32.update(encryptedData, 0, encryptedData.length);
            byte[] decryptedData = decrypt(encryptedData, encryptedData.length, keys);
            
            InputStream decryptedStream = new InputStream(decryptedData);
            int decompressedLength = decryptedStream.readInt();
            data = GZip.decompress(decryptedStream.getRemaining(), compressedLength);
            break;
        }
    }
    
    int revision = -1;
    if (stream.remaining() >= 2)
    {
        revision = stream.readUnsignedShort();
    }
    
    Container container = new Container(compression, revision);
    container.data = data;
    container.crc = crc32.getHash();
    return container;
}
```

<Note>
Some archives (particularly map regions) are encrypted with XTEA keys. Pass the keys array to decrypt them.
</Note>

### Item Loader Implementation

The `ItemLoader` parses binary item definitions using opcodes:

```java
// From ItemLoader.java:39-58
public ItemDefinition load(int id, byte[] b)
{
    ItemDefinition def = new ItemDefinition(id);
    InputStream is = new InputStream(b);
    
    while (true)
    {
        int opcode = is.readUnsignedByte();
        if (opcode == 0)
        {
            break;
        }
        
        this.decodeValues(opcode, def, is);
    }
    
    post(def);
    return def;
}
```

<Accordion title="Common Item Opcodes">
- **1**: Inventory model ID
- **2**: Item name
- **3**: Examine text
- **11**: Stackable flag
- **12**: Item cost/value
- **16**: Members item flag
- **30-34**: Ground menu options
- **35-39**: Inventory menu options
- **40**: Color replacements
- **97/98**: Noted item IDs
- **249**: Custom parameters (key-value pairs)
</Accordion>

```java
int hash = Djb2.hash("itemdefinitions");
```

```java
public Archive findArchiveByName(String name)
{
    int hash = Djb2.hash(name);
    for (Archive a : archives)
    {
        if (a.getNameHash() == hash)
        {
            return a;
        }
    }
    return null;
}
```

## Building a Cache Reader

Complete example of reading all items from the cache:

```java
import net.runelite.cache.IndexType;
import net.runelite.cache.definitions.ItemDefinition;
import net.runelite.cache.definitions.loaders.ItemLoader;
import net.runelite.cache.fs.*;

import java.io.File;
import java.util.HashMap;
import java.util.Map;

public class CacheReader
{
    public static void main(String[] args) throws Exception
    {
        // Open the cache
        Store store = new Store(new File("/path/to/cache"));
        store.load();
        
        // Get CONFIGS index
        Index configIndex = store.getIndex(IndexType.CONFIGS);
        
        // Items are in archive 10
        Archive itemArchive = configIndex.getArchive(10);
        
        // Load and decompress the archive
        Storage storage = store.getStorage();
        byte[] archiveData = storage.loadArchive(itemArchive);
        ArchiveFiles files = itemArchive.getFiles(archiveData);
        
        // Parse all item definitions
        ItemLoader loader = new ItemLoader();
        Map<Integer, ItemDefinition> items = new HashMap<>();
        
        for (FSFile file : files.getFiles())
        {
            ItemDefinition item = loader.load(file.getFileId(), file.getContents());
            items.put(item.id, item);
        }
        
        System.out.println("Loaded " + items.size() + " items");
        
        // Example: Print twisted bow stats
        ItemDefinition tbow = items.get(20997);
        System.out.println("Name: " + tbow.name);
        System.out.println("Examine: " + tbow.examine);
        System.out.println("Members: " + tbow.members);
        
        store.close();
    }
}
```

<Warning>
Cache reading is I/O intensive. Consider caching parsed definitions in memory or a database for production use.
</Warning>

## Best Practices

<Card title="Cache Library Usage">
- Always close the `Store` when finished to release file handles
- Cache parsed definitions to avoid repeated I/O
- Use try-with-resources for automatic resource management
- Handle missing archives gracefully (not all IDs exist)
- Verify CRC checksums when writing cache data
- Keep XTEA keys secure when working with map data
</Card>
